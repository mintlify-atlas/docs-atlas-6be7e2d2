## Overview

The `Span` struct represents a wire connection between two ports on poles. Spans are the edges in the wire network graph and carry electrical or communication conductors defined by an associated bundle.

## Structure Definition

```cpp
struct Span {
  ObjectId id = kInvalidObjectId;
  std::string display_id{};
  ObjectId port_a_id = kInvalidObjectId;
  ObjectId port_b_id = kInvalidObjectId;
  SpanKind kind = SpanKind::kGeneric;
  SpanLayer layer = SpanLayer::kUnknown;
  ObjectId bundle_id = kInvalidObjectId;
  ObjectId anchor_a_id = kInvalidObjectId;
  ObjectId anchor_b_id = kInvalidObjectId;
  ConnectionContext placement_context = ConnectionContext::kTrunkContinue;
  bool generated_by_rule = false;
  bool placement_override_flag = false;
  bool orientation_override_flag = false;
  double reference_length_m = 0.0;
  GenerationMeta generation{};
};
```

## Related Types

### SpanKind Enum

```cpp
enum class SpanKind : std::uint8_t {
  kGeneric = 0,
  kDistribution = 1,
  kService = 2,
};
```

Classifies spans by their functional role:
- **kGeneric**: Default/unspecified span type
- **kDistribution**: Primary distribution line between poles
- **kService**: Service drop to customer/building

### SpanLayer Enum

```cpp
enum class SpanLayer : std::uint8_t {
  kUnknown = 0,
  kHighVoltage = 1,
  kLowVoltage = 2,
  kCommunication = 3,
  kOptical = 4,
};
```

Defines infrastructure layers for span organization:
- **kUnknown**: Layer not specified
- **kHighVoltage**: High voltage power layer
- **kLowVoltage**: Low voltage power layer
- **kCommunication**: Communication/data layer
- **kOptical**: Optical fiber layer

### ConnectionContext Enum

```cpp
enum class ConnectionContext : std::uint8_t {
  kTrunkContinue = 0,
  kCornerPass = 1,
  kBranchAdd = 2,
  kDropAdd = 3,
};
```

Describes the topological role of the span:
- **kTrunkContinue**: Span continues a main trunk line
- **kCornerPass**: Span passes through a corner
- **kBranchAdd**: Span is a new branch splitting off
- **kDropAdd**: Span is a service drop

## Reference Length

The `reference_length_m` field is critical for maintaining consistent wire visualization:

- When poles are initially placed, the reference length is typically set to the geometric distance between ports
- This length is used for catenary sag calculations instead of the current geometric distance
- If poles tilt or move slightly, using the reference length keeps the visual wire tension stable
- Set to 0.0 to use the current geometric distance (dynamic calculation)

## Usage Notes

- Spans connect two [Ports](/api/entities/port) on (typically different) [Poles](/api/entities/pole)
- Each span references a [Bundle](/api/entities/bundle) that defines conductor count, spacing, and visual properties
- Optional anchor references (`anchor_a_id`, `anchor_b_id`) provide additional support modeling
- The span forms a directed edge from port_a to port_b, though electrically the connection is bidirectional
- Multiple spans can share the same bundle if they carry the same wire type
