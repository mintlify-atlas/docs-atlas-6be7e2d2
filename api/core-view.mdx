## Overview

`CoreView` provides read-only access to a `CoreState` instance. It exposes all entity stores, indexes, settings, and caches without allowing mutations. This is useful for querying state, rendering, serialization, and debugging.

**Location:** `wire/core/core_state.hpp:537-591`

**Namespace:** `wire::core`

## Usage Example

```cpp
#include "wire/core/core_state.hpp"

using namespace wire::core;

void render_network(const CoreState& state) {
  CoreView view = state.view();
  
  // Iterate over all poles
  for (const auto& [id, pole] : view.poles().all()) {
    // Render pole...
  }
  
  // Iterate over all spans
  for (const auto& [id, span] : view.spans().all()) {
    // Get cached curve points
    const auto* curve = view.find_span_visual_cache(id);
    if (curve) {
      for (const auto& part : curve->parts) {
        // Render visual part...
      }
    }
  }
  
  // Check recalc stats
  const auto& stats = view.last_recalc_stats();
  std::cout << "Last recalc processed " << stats.total_processed() 
            << " items" << std::endl;
}
```

## Obtaining a CoreView

Call `view()` on a `CoreState` instance:

```cpp
CoreState state;
// ... make edits ...

CoreView view = state.view();
```

**Source location:** `core_state.hpp:592`

```cpp
inline CoreView CoreState::view() const { 
  return CoreView(*this); 
}
```
