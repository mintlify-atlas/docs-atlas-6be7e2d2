## Overview

Wire is a game engine-independent C++ core library designed for generating, editing, and validating utility pole and power line networks. Built with real-world infrastructure simulation in mind, Wire provides a robust foundation for procedural generation of electrical distribution networks with automatic layout, connection management, and incremental recalculation.

## Architecture

Wire is built on a **4-layer architecture** that strictly separates concerns:

### Entity Layer (Persistent)

The authoritative data model that gets saved and loaded:

- **Pole** — Support structures (utility poles, towers)
- **Port** — Connection endpoints where wires attach
- **Anchor** — Support-only points (insulators, mounting hardware)
- **Bundle** — Multi-conductor wire groups with shared properties
- **Span** — Physical connections between two ports
- **Attachment** — Parametric objects along spans (dampers, spacers)

### Cache Layer (Derived)

Regenerable computed data not persisted:

- **CurveCache** — Sampled wire geometry with sag
- **BoundsCache** — Spatial bounding boxes for culling
- **VisualCache** — Support arm and insulator visual parts

### Runtime Layer (Transient)

Version tracking for incremental updates:

- **SpanRuntimeState** — Per-span version counters and dirty flags
- **DirtyQueue** — Work queues for geometry, bounds, and render updates

### Debug Layer (Session)

Non-authoritative diagnostic information:

- Path direction evaluation logs
- Slot selection decision records
- Lane assignment debug data

<Note>
The strict separation between persistent entity data and derived caches ensures data integrity and makes the system easy to serialize, debug, and extend.
</Note>

## Design Principles

Wire follows several core design principles:

- **Coordinate System** — Uses Unreal Engine conventions with double precision
- **ID Management** — 64-bit persistent IDs that are never reused
- **Edit API Only** — All modifications go through `CoreState` editing methods
- **Unidirectional Dirty Propagation** — Avoids circular dependencies in update chains
- **Slot vs. Port Distinction** — Template slots are candidates; ports are actual connection points

## Use Cases

Wire is designed for:

- **Procedural City Generation** — Automatically create realistic utility infrastructure
- **Infrastructure Simulation** — Model electrical distribution networks for games and training
- **Visualization Tools** — Build editing tools for power line layout and planning
- **Game Engine Integration** — Embed into Unreal Engine, Unity, or custom engines

## Implementation Status

Wire is currently in **Phase 4.x** (Design Consolidation):

- ✅ **Phase 0-4** — Core foundation, editing API, dirty tracking, geometry generation
- 🔄 **Phase 4.x** — Current: Design consolidation and type inventory
- 📋 **Phase 5+** — Planned: Serialization, raycasting, culling/LOD, wind simulation

<Info>
Wire is under active development. The core entity model and editing API are stable, but advanced features like serialization and culling are still in progress.
</Info>
